# GitHub 简介
## 第一节「GitHub 是什么？」

---

### **教案：GitHub 是什么？——代码界的魔法图书馆**  
**教学目标**：  
1. 用生活化比喻理解 GitHub 的核心功能  
2. 激发对开源协作的兴趣  
3. 完成第一个 GitHub 互动小任务  

---

#### **一、课堂导入：一个程序员的烦恼**  
**故事开场**（2分钟）：  
> “小明是个程序员，他写了一个超酷的计算器程序，但有一天电脑坏了，代码全丢了！  
> 他想找个地方备份代码，还想让朋友帮忙改进程序，该怎么办呢？”  

**提问互动**：  
“如果你是小明，你会怎么解决这个问题？”  
（学生自由回答，引导到“云存储”“协作工具”等方向）

---

#### **二、GitHub 的魔法世界**  
**1. 类比讲解：GitHub = 代码的魔法图书馆**（5分钟）  
- **图书馆 = 代码仓库**：存放代码的地方，谁都能来“借书”（下载）  
- **图书管理员 = Git**：记录每一本书的修改历史（版本控制）  
- **读者交流区 = Issues**：讨论书的 bug 和改进意见  
- **魔法传送门 = Pull Request**：读者可以提交自己的修改建议  

**视觉辅助**：  
展示 GitHub 界面截图，用表情包标注功能（如 📚=仓库，🔍=搜索，✨=Star）  

---

#### **三、动手时间：体验 GitHub 的“魔法”**  
**任务 1：探索一个神奇仓库**（10分钟）  
1. 打开 [GitHub 上的“恐龙游戏”仓库](https://github.com/wayou/t-rex-runner)  
2. 学生分组完成：  
   - 找到游戏的“启动按钮”（README 文件中的在线试玩链接）  
   - 在 Issues 区找一个有趣的用户提问（如“怎么让恐龙跳得更高？”）  
   - 观察仓库的 Star 数量（类比“点赞”）  

**任务 2：创建你的魔法档案**（15分钟）  
1. 学生注册 GitHub 账号（提前准备好邮箱）  
2. 完成个人主页的“魔法仪式”：  
   - 上传头像（建议用搞怪照片）  
   - 写一句个性签名（如“未来的代码巫师”）  
   - 关注老师账号（建立班级“魔法联盟”）  

---

#### **四、知识彩蛋：GitHub 的隐藏技能**  
**趣味冷知识**（3分钟）：  
- GitHub 上有 2 亿个仓库，相当于 200 个国家图书馆的藏书量！  
- 有人用 GitHub 存手绘漫画、菜谱甚至情书（展示 [Awesome GitHub](https://github.com/awesome) 奇葩项目）  
- 微软花 75 亿美元收购 GitHub，相当于买下 750 吨黄金！  

---

#### **五、课后任务：成为魔法学徒**  
1. **基础任务**：  
   - 找到一个让你心动的开源项目，点 ★ Star  
   - 在班级群分享你发现的“最奇葩 GitHub 仓库”  
2. **进阶挑战**：  
   - 修改自己的 GitHub 个人主页，添加一个emoji介绍自己（如 🐱=爱猫人士，🎮=游戏玩家）  

---

#### **六、教学反馈**  
**趣味问卷**（匿名在线表单）：  
1. 你觉得自己现在对 GitHub 的理解程度是？  
   - 🧙♂️ 魔法师级别  
   - 🧑🔬 实验室学徒  
   - 🧸 还在魔法学校门口  
2. 你希望下次课探索 GitHub 的哪个“魔法领域”？  
   - 隐藏副本（分支管理）  
   - 时空穿梭（版本回退）  
   - 魔法对决（解决代码冲突）  

---

**教学小贴士**：  
- 准备一些 GitHub 贴纸或徽章奖励积极参与的学生  
- 用“代码巫师帽”道具奖励最先完成任务的学生  
- 播放轻快的 BGM（推荐《哈利波特》主题曲）  

通过故事化场景和游戏化任务，学生能在笑声中建立对 GitHub 的直观认知，消除技术恐惧感！ 🎉


## 第二节「注册与设置 GitHub」

---

### **教案：成为代码世界的居民——注册与设置 GitHub**  
**教学目标**：  
1. 学会注册并设置一个个性化的 GitHub 账号  
2. 掌握 SSH 密钥配置的基本方法  
3. 激发对 GitHub 的兴趣，培养技术探索精神  

---

#### **一、课堂导入：欢迎来到代码星球！**  
**故事开场**（3分钟）：  
> “欢迎来到代码星球！这里是一个神奇的世界，每个居民都有自己的‘魔法档案’。今天，我们将帮助你们成为正式居民，并领取属于自己的‘身份令牌’。”  

**提问互动**：  
“如果让你设计自己的代码星球档案，你会希望它是什么样子？”  
（引导学生思考头像、昵称和个人简介的设计）

---

#### **二、第一步：注册你的“居民身份证”**  
**1. 注册 GitHub 账号**（5分钟）  
- 打开 [GitHub 官网](https://github.com/) 并点击 `Sign up`。  
- 引导学生完成以下步骤：  
  - 输入用户名（建议使用创意昵称，如 `CodeWizard123` 或 `BugHunter99`）。  
  - 填写邮箱地址（提醒使用常用邮箱以便接收通知）。  
  - 设置密码（强调安全性，推荐使用强密码生成器）。  

**趣味任务**：  
让学生用表情符号设计用户名，例如：  
- 🐱CoderCat  
- 🚀SpaceDev  

**2. 验证邮箱**  
- 提醒学生检查邮箱并点击验证链接。  
- 成功后宣布：“恭喜你，现在你是代码星球的正式居民了！”  

---

#### **三、第二步：打造个性化档案**  
**1. 上传头像**（5分钟）  
- 引导学生进入个人主页，点击头像区域上传图片。  
- 鼓励创意头像，例如：  
  - 自己的照片  
  - 喜欢的卡通形象  
  - 代表自己兴趣的图案（如猫咪、游戏手柄等）  

**2. 编辑个人资料**  
- **昵称**：可以是有趣的别名，如“代码巫师”或“Bug终结者”。  
- **个人简介**：鼓励学生用一句话介绍自己，例如：  
  - “正在学习 Python 的新手开发者 🐍”  
  - “热爱开源，喜欢解决难题 💻✨”  
- **添加 emoji**：让简介更生动，例如：  
  - 🎮 游戏开发者  
  - 🎨 前端设计师  

**趣味互动**：  
让学生互相猜对方的简介含义，比如看到 🎵 可能是音乐爱好者。

---

#### **四、第三步：领取“身份令牌”（SSH 密钥配置）**  
**1. 什么是 SSH 密钥？**（3分钟）  
- **类比讲解**：SSH 密钥就像一把“魔法钥匙”，用来安全地连接你的电脑和 GitHub。  
- 展示 SSH 密钥的作用：无需每次输入密码即可推送代码。  

**2. 生成 SSH 密钥**（10分钟）  
- 在终端中运行以下命令：
  ```bash
  ssh-keygen -t ed25519 -C "你的邮箱地址"
  ```
  （如果没有 `ed25519` 支持，可以用 `rsa` 替代：`ssh-keygen -t rsa -b 4096 -C "你的邮箱地址"`）  

- 提醒学生选择默认路径保存密钥文件，并设置一个简单但安全的密码短语（passphrase）。  

**3. 添加 SSH 密钥到 GitHub**  
- 打开生成的公钥文件（通常是 `~/.ssh/id_ed25519.pub`），复制内容。  
- 进入 GitHub 设置页面 -> `SSH and GPG keys` -> 点击 `New SSH key`，粘贴公钥并命名（如“My Laptop Key”）。  

**趣味任务**：  
给 SSH 密钥起个名字，例如：  
- “星际传送门”  
- “代码守护者”  

---

#### **五、第四步：测试你的“魔法钥匙”**  
**1. 测试连接**  
在终端运行以下命令，确保 SSH 配置成功：
```bash
ssh -T git@github.com
```
- 如果看到类似以下消息，则表示成功：
  > Hi 用户名! You've successfully authenticated, but GitHub does not provide shell access.

**2. 庆祝仪式**  
宣布：“恭喜你，现在你已经拥有了通往代码星球的魔法钥匙！”  

---

#### **六、知识彩蛋：GitHub 的隐藏技能**  
**趣味冷知识**（3分钟）：  
- GitHub 上有超过 1 亿用户，相当于地球人口的 1%！  
- 有人用 GitHub 存储整个国家的法律条文（如德国的《民法典》）。  
- GitHub 的吉祥物是一只名叫 **Octocat** 的八爪猫 🐙🐱。  

---

#### **七、课后任务：完善你的魔法档案**  
1. **基础任务**：  
   - 在个人主页添加至少 3 个 emoji 表情符号，描述你的兴趣爱好。  
   - 关注班级同学的 GitHub 账号，建立“魔法联盟”。  

2. **进阶挑战**：  
   - 创建一个空仓库，命名为 `my-first-repo`，并在 README 文件中写下一句自我介绍。  
   - 尝试将仓库设置为公开或私有，观察区别。  

---

#### **八、教学反馈**  
**趣味问卷**（匿名在线表单）：  
1. 你觉得今天的课程难度如何？  
   - 🎢 太简单了，还想学更多！  
   - 🎠 刚刚好，学会了新东西！  
   - 🎡 有点难，需要再复习一下。  
2. 你最喜欢哪个环节？  
   - 注册账号  
   - 设置头像和简介  
   - 配置 SSH 密钥  

---

**教学小贴士**：  
- 准备一些 GitHub 主题的小礼物（如贴纸、徽章）奖励积极参与的学生。  
- 用 Octocat 表情包作为课堂装饰，增加趣味性。  
- 播放轻松的背景音乐（如《Stardew Valley》原声带）营造氛围。  

通过故事化场景和互动任务，学生不仅能快速掌握 GitHub 的注册与设置，还能感受到技术学习的乐趣！ 🎉
